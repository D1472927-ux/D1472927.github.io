# 產品需求文件（PRD）— 線上算命系統

> **文件版本：** v1.0
> **建立日期：** 2026-04-09
> **技術棧：** Flask + Jinja2 + SQLite

---

## 1. 專案概述

### 1.1 背景與動機

算命、占卜、抽籤是華人文化中歷史悠久的傳統活動，許多人在面臨人生重大決定或日常困惑時，會前往廟宇求籤問卜。然而，現代人生活忙碌，不一定有時間親自前往廟宇。本系統旨在將傳統的算命、占卜、抽籤體驗搬到線上，讓使用者隨時隨地都能透過網頁進行問卜，獲得心靈上的指引與慰藉。

### 1.2 目標用戶

| 用戶類型 | 說明 |
|---------|------|
| **一般民眾** | 對算命、占卜有興趣，希望在線上方便地進行抽籤與占卜的使用者 |
| **年輕族群** | 習慣使用網路服務，想透過趣味方式了解運勢的學生或上班族 |
| **信仰群眾** | 平時有拜拜、求籤習慣，但無法經常前往廟宇的信眾 |
| **管理者（未來擴充）** | 管理籤詩內容、查看捐款紀錄的系統管理人員 |

### 1.3 核心價值主張

- **便利性**：打破時間與空間限制，隨時隨地都能線上求籤問卜
- **趣味性**：結合多種占卜方式（抽籤、塔羅、星座運勢等），增加互動樂趣
- **紀錄性**：自動儲存每次算命結果，方便回顧與對照
- **文化傳承**：將傳統算命文化以數位化方式保存與推廣

## 2. 功能需求

### 功能一：線上抽籤（求籤）

**使用者故事：**
> 作為一位**一般使用者**，我希望能**在線上進行抽籤（求籤）**，以便**在遇到困惑時獲得籤詩指引，不需親自前往廟宇**。

**功能描述：**
- 使用者可選擇想詢問的類別（如：事業、感情、學業、健康、財運）
- 輸入想問的問題（選填）
- 點擊「抽籤」按鈕後，系統隨機產生一支籤（上上籤、上籤、中籤、下籤、下下籤）
- 顯示對應的籤詩內容與白話文解釋
- 籤詩資料至少包含 30 支以上

---

### 功能二：塔羅占卜

**使用者故事：**
> 作為一位**對西方占卜有興趣的使用者**，我希望能**進行線上塔羅牌占卜**，以便**透過不同的占卜方式獲得多元的指引**。

**功能描述：**
- 使用者選擇占卜主題（如：今日運勢、感情問題、工作抉擇）
- 從牌陣中翻開指定數量的塔羅牌（如：單牌占卜、三牌占卜）
- 顯示抽到的塔羅牌圖片、牌義解釋與建議
- 支援正位與逆位的不同解讀

---

### 功能三：每日運勢

**使用者故事：**
> 作為一位**每天想了解運勢的使用者**，我希望能**查看今日運勢**，以便**提前了解今天的整體運勢，做好心理準備**。

**功能描述：**
- 使用者登入後可查看今日運勢
- 運勢內容涵蓋：整體運勢、愛情運、事業運、財運、幸運色、幸運數字
- 每日運勢根據日期自動更新
- 以星級（1~5 星）方式呈現各面向運勢

---

### 功能四：算命結果儲存與歷史紀錄

**使用者故事：**
> 作為一位**經常使用本系統的使用者**，我希望能**儲存每次的算命結果並查看歷史紀錄**，以便**日後回顧與對照結果是否靈驗**。

**功能描述：**
- 使用者需註冊 / 登入帳號才能使用儲存功能
- 每次算命結果自動記錄（類型、時間、問題、結果）
- 提供「我的紀錄」頁面，以時間軸方式呈現歷史紀錄
- 支援依照日期、類型篩選紀錄
- 使用者可刪除不需要的紀錄

---

### 功能五：線上捐香油錢

**使用者故事：**
> 作為一位**心懷感恩的使用者**，我希望能**透過系統捐獻香油錢**，以便**表達對神明的敬意與感謝**。

**功能描述：**
- 提供捐獻頁面，使用者可選擇捐獻金額（預設選項 + 自訂金額）
- 使用者可選擇是否留下姓名或匿名捐獻
- 可填寫祈願內容（選填）
- 捐獻後顯示感謝頁面與捐獻記錄
- 提供「我的捐獻紀錄」查詢功能

> ⚠️ **注意：** MVP 階段僅模擬捐獻流程（不串接真實金流），記錄捐獻意向與金額即可。

---

### 功能六：使用者帳號系統

**使用者故事：**
> 作為一位**使用者**，我希望能**註冊並登入帳號**，以便**儲存我的算命紀錄與捐獻紀錄，獲得個人化體驗**。

**功能描述：**
- 支援帳號註冊（使用者名稱 + 密碼）
- 支援登入 / 登出功能
- 個人頁面顯示基本資訊、算命次數統計、捐獻總額
- 密碼以雜湊方式安全儲存

---

### 功能七：籤詩分享功能

**使用者故事：**
> 作為一位**覺得籤詩很有意義的使用者**，我希望能**將算命結果分享出去**，以便**與朋友分享有趣或有啟發的籤詩**。

**功能描述：**
- 算命結果頁面提供「分享」按鈕
- 可複製結果連結（產生一個專屬的結果分享頁面）
- 分享頁面以美觀的卡片格式呈現籤詩內容

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| **後端框架** | Flask（Python） |
| **模板引擎** | Jinja2（Flask 內建） |
| **資料庫** | SQLite（輕量級，免安裝） |
| **前端** | HTML + CSS + JavaScript（Jinja2 模板渲染） |
| **部署環境** | 本地開發環境（localhost） |

### 3.2 效能考量

- 頁面載入時間應在 **2 秒以內**
- 抽籤 / 占卜的隨機結果應即時回應（< 500ms）
- SQLite 資料庫足以應付開發階段的資料量

### 3.3 安全考量

- 使用者密碼必須以 **bcrypt 或 werkzeug 雜湊** 方式儲存，禁止明文儲存
- 所有表單輸入須進行 **輸入驗證與防止 SQL Injection**
- 使用 Flask 的 **CSRF 保護** 機制
- 登入狀態使用 Flask Session 管理

### 3.4 使用者介面

- 採用 **中式神秘風格** 的視覺設計（紅色、金色、深色系為主色調）
- 支援 **響應式設計（Responsive Design）**，在手機與桌機都能正常使用
- 使用 **Google Fonts 中文字型**（如 Noto Serif TC）提升質感
- 加入適當的 **動畫效果**（如抽籤翻牌動畫）提升互動體驗

## 6. 頁面清單（初步規劃）

| 頁面名稱 | 路徑 | 說明 |
|---------|------|------|
| 首頁 | `/` | 系統入口，展示主要功能入口與今日運勢摘要 |
